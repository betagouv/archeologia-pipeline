================================================================================
                    TESTS MANUELS - PLUGIN ARCHEOLOG'IA PIPELINE
================================================================================

Ces tests doivent être exécutés dans QGIS avec le plugin chargé.
Les tests automatisés (pytest) couvrent la logique métier hors QGIS :
    python run_tests.py

Pré-requis :
- QGIS 3.34+ installé
- Plugin activé dans QGIS
- PDAL, GDAL (gdalwarp, gdal_translate, gdaladdo) dans le PATH
- Runner ONNX compilé dans third_party/cv_runner_onnx/<os>/
- Au moins un modèle ONNX dans models/

================================================================================
1. CHARGEMENT DU PLUGIN
================================================================================

[ ] 1.1 Démarrage QGIS
    - Ouvrir QGIS
    - Vérifier qu'aucune erreur n'apparaît dans la console Python
    - Vérifier que le plugin apparaît dans le menu Extensions

[ ] 1.2 Ouverture du plugin
    - Cliquer sur le menu/icône du plugin
    - Vérifier que MainDialog s'ouvre sans erreur
    - Vérifier que tous les onglets sont visibles

[ ] 1.3 Fermeture et réouverture
    - Fermer puis rouvrir le plugin
    - Vérifier que la configuration précédente est conservée

================================================================================
2. INTERFACE UTILISATEUR
================================================================================

[ ] 2.1 Changement de mode
    - Sélectionner chaque mode (ign_laz, local_laz, existing_mnt, existing_rvt)
    - Vérifier que les champs spécifiques au mode s'affichent
    - Vérifier que le label du champ source change selon le mode

[ ] 2.2 Sélection du dossier de sortie
    - Sélectionner un dossier valide → le chemin s'affiche

[ ] 2.3 Configuration des produits
    - Cocher/décocher chaque produit (MNT, DENSITE, M_HS, SVF, SLO, LD, VAT)

[ ] 2.4 Configuration des paramètres RVT
    - Modifier les paramètres de chaque onglet RVT (MDH, SVF, Slope, LDO, VAT)

[ ] 2.5 Configuration Computer Vision
    - Activer/désactiver la CV
    - Sélectionner un modèle ONNX dans models/
    - Modifier les seuils (confidence, IOU)

================================================================================
3. SAUVEGARDE / CHARGEMENT CONFIG
================================================================================

[ ] 3.1 Sauvegarde manuelle → fermer/rouvrir → paramètres conservés
[ ] 3.2 Reset config générale → valeurs par défaut restaurées
[ ] 3.3 Reset config RVT → valeurs RVT par défaut restaurées
[ ] 3.4 Pas de sauvegarde auto sur Run → modifier, Run, Cancel, rouvrir → non sauvé

================================================================================
4. PREFLIGHT (run_preflight)
================================================================================

[ ] 4.1 Dépendances OK → preflight passe sans erreur
[ ] 4.2 PDAL manquant → erreur affichée dans les logs (mode ign_laz)
[ ] 4.3 Chemin d'entrée invalide → erreur "introuvable" dans les logs
